# 日志查看器

Super Client R 提供独立窗口的日志查看器，帮助开发者和高级用户查看和分析应用运行日志。

## 打开日志查看器

日志查看器以独立窗口运行，可以通过以下方式打开：

- 在设置中点击 **打开日志查看器**
- 通过菜单栏进入

## 日志存储

日志使用 SQLite 数据库（better-sqlite3）存储在本地：

- **存储位置**：`{用户数据目录}/logs/logs.db`
- **最大容量**：自动维护不超过 100,000 条记录
- **存储模式**：WAL（Write-Ahead Logging）模式，支持并发读写

每条日志记录包含：

- 时间戳
- 日志级别（debug、info、warn、error 等）
- 模块名称
- 进程类型（main、renderer 等）
- 消息内容
- 元数据信息
- 错误详情（如有）
- 会话 ID

## 查看与筛选

### 筛选条件

日志查看器支持多种筛选方式：

- **按级别** - 选择显示特定级别的日志（debug、info、warn、error）
- **按模块** - 筛选特定模块产生的日志
- **按进程** - 区分主进程和渲染进程的日志
- **关键词搜索** - 在消息内容中搜索关键词
- **时间范围** - 设置开始和结束时间进行范围查询

### 排序与分页

- 支持按时间正序或倒序排列
- 默认每页 50 条记录
- 可配置每页显示数量

## 统计面板

日志查看器包含统计面板，提供：

- **总日志数量** - 数据库中的总记录数
- **各级别数量** - 按日志级别分类统计
- **各模块数量** - 按模块分类统计
- **各进程数量** - 按进程类型分类统计
- **近 24 小时错误数** - 最近一天的错误数量
- **时间分布** - 按小时统计的日志数量直方图

## 日志详情

点击日志列表中的任意一条记录，右侧抽屉会展示该日志的完整详情，包含所有元数据和错误堆栈信息。

## 导出与清除

- **导出日志** - 将当前筛选结果导出到文件
- **清除日志** - 清空整个日志数据库
