# pr-reviewer — PR 审查助手

基于项目架构规范审查代码变更，覆盖 IPC 模板合规、进程边界、类型安全、i18n 完整性、安全性等 10 个维度。

## 安装

将 `skills/pr-reviewer` 目录放置在项目的 `skills/` 下，Claude Code 会自动加载。

## 使用方式

### 斜杠命令

```
/review
```

### 自然语言触发

- "审查 PR"
- "review changes"
- "检查变更"
- "代码审查"

## 功能

- 自动读取 git diff 获取完整变更
- 10 维度审查清单（架构/类型/React/i18n/样式/安全/错误/命名/性能/质量）
- 验证 IPC 6 步模板完整性
- 检测反模式（Node API 入侵 Renderer、同步 IPC、内存泄漏等）
- 三级严重度（❌ 必须修复 / ⚠️ 建议修复 / 💡 可以改进）
- 不仅找问题，也肯定做得好的地方

## 与 code-reviewer 的区别

| 维度 | code-reviewer | pr-reviewer |
|------|--------------|-------------|
| 粒度 | 代码片段级别 | PR / 变更集级别 |
| 关注点 | 通用代码质量 | 项目特定规范 + 架构 |
| 规范基线 | 通用最佳实践 | CLAUDE.md 中定义的规范 |
